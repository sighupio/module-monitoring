# Monitoring Module Release v4.2.0

Welcome to the latest release of the `monitoring` core module of the [`SIGHUP Distribution`](https://github.com/sighupio/distribution), maintained by team SIGHUP by ReeVo.

This release adds the support to Kubernetes 1.35, drops support for Kubernetes 1.31, ships a major upgrade of the monitoring stack aligned with [kube-prometheus v0.17.0](https://github.com/prometheus-operator/kube-prometheus/releases/tag/v0.17.0), and introduces the new `x509-exporter` v4 package layout generated from the upstream Helm chart.

## Bug Fixes 🐞

- [[#220](https://github.com/sighupio/module-monitoring/pull/220)] Updated the deprecated `etcd_debugging_mvcc_db_total_size_in_bytes` Prometheus metric to the promoted `etcd_mvcc_db_total_size_in_bytes` to fix the "Database Size" panel in the etcd Grafana dashboard.
- [[#050044c](https://github.com/sighupio/module-monitoring/commit/050044c)] Fixed wrong CoreDNS dashboard metrics after the kube-prometheus `v0.17.0` upstream bump.
- [[#c8eaee7](https://github.com/sighupio/module-monitoring/commit/c8eaee7)] Forced `kube-state-metrics` to expose all default resources (including `endpoints`) via the `--resources` flag. Required by the `MimirGossipMembersEndpointsOutOfSync` alert in the `mimir` package (see `katalog/mimir/MAINTENANCE.md`).
- [[#ee2b21d](https://github.com/sighupio/module-monitoring/commit/ee2b21d)] Added the missing RBAC permissions for `EndpointSlice` discovery to the `prometheus-k8s` `ClusterRole` in `prometheus-operated`.
- [[#e9b4bb2](https://github.com/sighupio/module-monitoring/pull/242)] Updated `x509-exporter` watch paths to use `kubelet-server-current.pem` and `kubelet-client-current.pem` instead of the legacy `kubelet.crt` file.
- [[#2c33b03](https://github.com/sighupio/module-monitoring/pull/241)] Updated `x509-exporter` `securityContext` on the control-plane DaemonSet to include the `DAC_READ_SEARCH` Linux capability, so the exporter can read etcd certificates on on-prem clusters where the on-prem installer restricts permissions on `/etc/etcd/pki` to the `etcd` user. Fixes [[#221](https://github.com/sighupio/module-monitoring/issues/221)].
- [[#c3900b3](https://github.com/sighupio/module-monitoring/commit/c3900b3)] Enabled the `exposeNotBefore` metric in the `x509-exporter` Helm values.
- [[#d742643](https://github.com/sighupio/module-monitoring/commit/d742643)] Replaced the broken `X509ExporterReadErrors` alert shipped with `x509-exporter` chart `v4.1.0` (which references the removed `x509_read_errors` metric) with the fixed `SourceErrors`/`SourceErrorsSustained` alerts and the related Grafana dashboard backported from `v4.2.0-rc.1`. Fixes [[sighupio/distribution#512](https://github.com/sighupio/distribution/issues/512)].

## Deprecations ⚠️

- **`prometheus-operated` — remote-write metrics:** the following Prometheus 3.x remote-write metrics are deprecated and will be removed in a future upstream release: `prometheus_remote_storage_samples_in_total`, `prometheus_remote_storage_histograms_in_total`, `prometheus_remote_storage_exemplars_in_total`, `prometheus_remote_storage_highest_timestamp_in_seconds`.

- **`kube-state-metrics` — `endpoints` resource:** starting with `v2.18.0`, `endpoints` is no longer in the default `--resources` list. We have explicitly re-enabled it via `--resources=…,endpointslices,…,endpoints` to preserve `kube_endpoint_address`, required by the `MimirGossipMembersEndpointsOutOfSync` alert. No breaking change for SD users.

## Update Guide 🦮

The furyctl tool now manages Module installations and upgrades. The instructions below are left for reference when using the legacy version of furyctl.

### Process

To upgrade this core module from `v4.1.0` to `v4.2.0`, execute the following:

```bash
kustomize build <your-project-path> | kubectl apply -f - --server-side
```
