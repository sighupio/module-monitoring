# MinIO HA - maintenance

Before running the script, check available versions:

1. Check the latest MinIO version: https://github.com/chainguard-forks/minio/releases
2. Check the latest MC version: https://github.com/minio/mc/releases
3. Check the latest Prometheus rules version: https://github.com/samber/awesome-prometheus-alerts/releases

Then run the upgrade script:

```bash
MINIO_VERSION=RELEASE.2026-05-20T23-44-52Z MC_VERSION=RELEASE.2025-08-13T08-35-41Z PROMETHEUS_ALERTS_VERSION=2026-04-10.1 ./upgrade.sh
```

The script automatically:

1. Fetches the specified release from [chainguard-forks/minio](https://github.com/chainguard-forks/minio/releases)
2. Fetches the specified mc release from [minio/mc](https://github.com/minio/mc/releases)
3. Downloads and extracts the Helm chart
4. Updates image tags in `MAINTENANCE.values.yaml` and `kustomization.yaml`
5. Re-renders `deploy.yaml` (excluding the ConfigMap, which is managed via kustomize)
6. Downloads upstream Prometheus rules and scopes all metrics to `{job="minio-monitoring"}`
7. Sets `for: 5m` on the `MinioDiskSpaceUsage` alert (upstream defaults to `0m`)
8. Runs `mise run add-license`

## Customizations

We diverge from the upstream Helm chart in the following ways:

1. **Affinity**: We use `preferredDuringSchedulingIgnoredDuringExecution` for pod anti-affinity to allow better scheduling flexibility in production environments.
2. **Secrets**: We use `existingSecret` to reference a secret generated by kustomize (`secretGenerator` in `kustomization.yaml`) instead of relying on Helm to create a secret from values. This allows us to manage the secret via kustomize and apply standard security practices (like external secrets).
3. **Image registry**: We point to our internal registry (`registry.sighup.io/fury/`) instead of the upstream registries.
4. **Init job**: We added a custom init job (`initialize-minio-buckets.yaml`) to create the `mimir` bucket after MinIO is ready.
5. **Prometheus rules**: We scope all metrics in the upstream rules to `{job="minio-monitoring"}` so alerts only fire for this MinIO instance.
